# Contents Hub 仕様概要

## 概要

Contents Hub は、自分が見た本、ゲーム、映像作品を記録し、あとから横断的に振り返れる個人向けのコンテンツ管理システムです。

本だけ、ゲームだけ、映像作品だけを個別に管理するのではなく、異なるメディアをひとつのサービスで統合管理できることを重視します。

## 目的

- 自分が触れたコンテンツを一元管理する
- 本、ゲーム、映像作品を媒体ごとに分断せずに記録する
- タイトルや作品情報を外部 API から取得して登録負荷を下げる
- 映像作品については配信状況も把握できるようにする

## 管理対象

### 本

- 書籍タイトル
- 著者名
- 書影
- 商品説明などの基本メタデータ

### ゲーム

- ゲームタイトル
- 対応ハードや関連情報
- パッケージ画像などの基本メタデータ

### 映像作品

- 映画、ドラマなどの作品情報
- ポスター画像や作品概要
- どのサブスクリプションサービスで配信されているかの情報

## 外部 API

### 楽天ブックス Developer API

本とゲームの情報取得に使用します。

想定用途:

- タイトル検索
- 書影や商品画像の取得
- 著者名、説明文、商品 URL などの取得

### TMDB API

映像作品の情報取得に使用します。

想定用途:

- 映画、ドラマなどのタイトル検索
- ポスター画像や概要の取得
- 作品の基本メタデータ取得

### Watchmode API

映像作品の配信状況取得に使用します。

想定用途:

- 配信中のサブスクリプションサービス確認
- 映像作品ごとの視聴可能環境の把握

## システム方針

- メディア種別ごとに情報源は異なるが、アプリ上はひとつのコンテンツとして扱えるようにする
- 登録時は手入力を最小化し、外部 API を使った検索と候補選択を中心にする
- 後から一覧・検索・振り返りがしやすいデータ構造を前提にする
- 映像作品は作品情報と配信情報を分けて管理し、更新しやすくする

## 技術構成

### バックエンド

- Supabase

用途:

- データベース
- 認証
- API 基盤
- 将来的なストレージやサーバーサイド処理の実装基盤

### フロントエンド

- Cloudflare Workers

用途:

- フロントエンドの配信
- サーバーサイドレンダリング・API エンドポイントの実行基盤
- エッジでの高速レスポンス

### 開発ツール

- パッケージマネージャー: pnpm
- 言語: TypeScript
- ビルドツール: Vite
- Linter: ESLint
- Formatter: Prettier
- バージョン管理ツール: mise
- IaC: Terraform
